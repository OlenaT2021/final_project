Glossary

  • Accessible rich internet applications (ARIA)
    A set of HTML roles and attributes developed by the World Wide Web Consortium (W3C). It aims to enhance the accessibility of web content and web-based applications for members of the disability community.
  • Cascading style sheets (CSS)
    A style sheet programming language that helps configure and manage the appearance and formatting of a document created in a markup language.
  • Component content management system (CCMS)
    A content management system that manages content at a granular level (component) rather than at the document level. Each component represents a single topic, concept or asset (for example, an image, table, product description, or a procedure).
  • Computer-assisted translation (CAT) tools
    Are software programs that assist human translators in transferring the meaning of text between languages more quickly and efficiently. More specifically, CAT tools provide translators with an interface for submitting, editing, managing, and storing multilingual content in one place.
  • Darwin Information Typing Architecture (DITA)
    A set of document types for authoring and organizing topic-oriented information, as well as a set of mechanisms for combining, extending, and constraining document types.
  • Document type definition (DTD)
    A structured description of the structure and content of an XML document.
  • Extensible markup language (XML)
    A markup language and file format for storing, transmitting, and reconstructing data. It defines a set of rules for encoding documents in a format that is both human-readable and machine-readable.
  • eXtensible Stylesheet Language (XSLT)
    An XML-based stylesheet language to present, style and extract fragments of XML documents.
  • Globalization (G11N)
    The process by which businesses or other organizations develop international influence or start operating on a global scale.
  • Hypertext Markup Language 5 (HTML5)
    A markup language used for structuring and presenting hypertext documents on the World Wide Web.
  • Input method editor (IME)
    A program that provides a specialized user interface for text input.
  • Internationalization (I18N)
    The design and development of a product, application or document content that enables easy Localization for target audiences that vary in culture, region, or language.
  • Localization (L10N)
    The linguistic adaptation of content for the target region, its languages, and cultural particularities to make it appropriate, easily understandable, and comfortable to use for local native language speakers.
  • Markdown
    A lightweight markup language created by John Gruber in 2004. It is used for adding formatting elements to plaintext documents. It is popular due to its readability, portability, and broad application support.
  • Microsoft Word
    A word processing program that allows for the creation of both simple and complex documents.
  • PDF/UA standard (formally known as ISO 14289-1)
    A standard designed to ensure that PDFs are universally accessible. Aimed at everyone involved in creating PDFs, it sets minimum requirements that make sure documents are compliant with devices that support people with disabilities.
  • Portable document format (PDF)
    A file format developed by Adobe in 1992 to present documents, including text formatting and images, in a manner independent of application software, hardware, and operating systems. It is used when there is a need to save files that cannot be modified but still need to be easily shared and printed.
  • Project manager (PM)
    A professional charged with overseeing the process of planning, executing and delegating responsibilities related to an organization's information technology (IT) pursuits and goals.
  • Quality assurance (QA)
    Any systematic process of determining whether a product or service meets specified requirements. QA establishes and maintains set requirements for developing or manufacturing reliable products.
  • Right-to-left (RTL)
    Languages written and read from right to left, top to bottom.
  • Transclusion
    In computer science, it is the inclusion of part or all of an electronic document into one or more other documents by reference via hypertext. Transclusion is usually performed when the referencing document is displayed, and is normally automatic and transparent to the end user.
  • Translation (T9N)
    The literal word-for-word transformation of a source text into the target language.
  • Translation management systems (TMS)
    A software that automates repetitive or manual tasks to make the translation process as efficient as possible.
  • User acceptance testing (UAT)
    A phase of software development in which the software is tested in the real world by its intended audience. It is also called application testing or end-user testing.
  • User interface (UI)
    Anything a user may interact with to use a digital product or service. This includes everything from screens and touchscreens, keyboards, sounds, and even lights.
  • Web accessibility initiative (WAI)
    An effort by the W3C to develop standards and guidelines to help everyone building the web ensure and improve accessibility for people with various challenges.
  • Web content accessibility guide (WCAG)
